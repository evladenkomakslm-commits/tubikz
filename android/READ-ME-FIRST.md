# ₮ubikz · Android (APK)

Внутри этой папки — готовый Android Studio проект (Capacitor).
Приложение это «нативная обёртка» вокруг сайта `https://tubikz.onrender.com`.

## ✅ Что важно знать про сервер

**Сервер на Render продолжает работать как обычно.**

- APK не содержит копию backend-а. Он показывает живой сайт через Android-овский WebView.
- БД (Neon), хранилище (UploadThing), Socket.io (Render) — всё то же самое.
- Когда ты деплоишь обновление через `./deploy.sh` — пользователи Android
  видят его **сразу же** при следующем открытии приложения, без переcборки APK.
- Учётки и переписки одни и те же между web и Android.

## 🔧 Открыть в Android Studio

1. **File → Open** → выбрать эту папку (`android/`).
2. Android Studio попросит установить недостающее:
   - **Android SDK Platform 35** (или последний)
   - **Build Tools 35.x**
   - **Gradle JDK 17** *(не 8 и не 11 — иначе билд упадёт)*
3. Дождаться окончания **Gradle Sync** (нижняя строка статуса). 5–10 минут на первом запуске.

## 🏗 Собрать APK

### Debug (для теста)
1. Подключи Android-телефон через USB, включи **USB Debugging** в настройках разработчика.
2. В Android Studio выбери устройство в выпадашке сверху.
3. Жми зелёный треугольник ▶ — соберёт debug APK и установит на телефон.

### Release (для распространения)
1. **Build → Generate Signed Bundle / APK → APK**
2. Создай новый Keystore (запомни пароль и `.jks` файл — он нужен для всех будущих обновлений!).
3. Выбери `release` вариант.
4. Нажми Finish — APK будет в `android/app/release/app-release.apk`.

## 🧪 Что должно работать

- ✅ Регистрация / вход (та же БД, что и на сайте)
- ✅ Чаты с real-time
- ✅ Голосовые сообщения (Android попросит разрешение на микрофон)
- ✅ Фото / видео из галереи
- ✅ Звонки (голосовые и видео) — **попросит микро + камеру при первом звонке**
- ✅ PWA-функции

## ⚠️ Известные нюансы Android-WebView

- На первом звонке Android **попросит разрешения** для камеры и микрофона.
  Если запретил — ничего не починится, пока не дашь руками в **Настройки → Apps → ₮ubikz → Permissions**.
- Если у тебя на телефоне очень старый Android (<8) — Capacitor требует Android 6+ (API 24+).
- WebRTC P2P может не пробиться через мобильный оператор с симметричным NAT (~10% сетей).
  Лечится TURN-сервером.

## 📦 Подмена URL (если переехал на другой хостинг)

В файле `../capacitor.config.ts` поменяй `server.url`, потом:

```bash
npx cap sync android
```

И пересобери APK в Android Studio.

## 🔄 Обновления

Просто пуш в Render, как обычно:
```bash
./deploy.sh "что-то"
```

Пользователи Android получают обновлённый веб-код моментально — APK перебирать не нужно.

APK нужно пересобирать только если:
- Меняется capacitor.config.ts (URL, app id, имя)
- Меняются иконки или splash
- Добавляются новые нативные плагины Capacitor
- Меняются разрешения в AndroidManifest.xml
